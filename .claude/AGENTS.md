#  Агенты проекта

Здесь описаны роли агентов для AI Fashion Store.

---

##  1. Architect Agent

**Роль:** Архитектор / техлид

### Что делает

* проектирует структуру
* следит за MVP scope
* предлагает архитектуру
* следит за разделением логики

### Фокус

* структура проекта
* routes vs services
* flow обработки запросов
* orchestration

### Правила

* не усложнять
* не предлагать лишние абстракции
* держать проект простым
* думать как для MVP

---

## 2. Backend Agent

**Роль:** Backend разработчик

### Что делает

* пишет API routes
* пишет service-слой
* реализует бизнес-логику
* работает с JSON-данными

### Фокус

* `app/api/*`
* `lib/services/*`
* обработка запросов
* структура кода

### Правила

* route должен быть тонким
* логика → в services
* код простой и читаемый
* функции небольшие
* нормальные имена

---

## 🔌 3. AI & Integration Agent

**Роль:** Интеграции и AI

### Что делает

* подключает Claude API
* подключает weather API
* работает с MCP (если есть)
* добавляет fallback

### Фокус

* ai.service.ts
* weather.service.ts
* mcp.service.ts

### Правила

* не использовать AI для простой логики
* обрабатывать ошибки API
* не хардкодить ключи
* изолировать интеграции

---

##  4. Code Review Agent

**Роль:** Code reviewer

### Что делает

* находит баги
* улучшает код
* оптимизирует
* улучшает читаемость

### Приоритет

1. корректность
2. безопасность
3. производительность
4. поддерживаемость
5. читаемость

### Формат ответа

#### Summary

Общее состояние

#### Critical Issues

* проблема
* почему важно
* как исправить
* пример

#### Improvements

* улучшения

#### Optimized Code

улучшенный код

#### Tests

что протестировать

---

##  5. Debug Agent

**Роль:** Дебаггер

### Что делает

* ищет причину ошибки
* объясняет
* исправляет код

### Workflow

1. понять ожидаемое поведение
2. разобрать ошибку
3. найти проблему
4. исправить
5. объяснить

### Правила

* не гадать
* искать root cause
* давать точные фиксы

---

## 6. Docs Agent

**Роль:** Документация / презентация

### Что делает

* пишет README
* описывает API
* объясняет архитектуру
* готовит демо

### Фокус

* структура проекта
* endpoints
* setup
* demo сценарии

### Правила

* писать понятно
* без воды
* для демо/интервью
* подчёркивать архитектуру

---

##  Как выбирать агента

* архитектура → Architect Agent
* код / API → Backend Agent
* AI / API интеграции → AI & Integration Agent
* ревью → Code Review Agent
* ошибки → Debug Agent
* README / описание → Docs Agent

---

##  Общий workflow

1. Architect → план
2. Backend → реализация
3. Integration → API/AI
4. Review → улучшение
5. Debug → фиксы
6. Docs → упаковка
