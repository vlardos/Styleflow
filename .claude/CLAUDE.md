##  Контекст проекта

Это прототип магазина одежды с AI-ассистентом.

* работать с каталогом товаров

* роутить запросы
* рекомендовать одежду
* учитывать погоду
* оформлять mock-заказы
* (опционально) использовать MCP

##  Главные правила

### 1. Не усложняй

* выбирай самое простое рабочее решение
* избегай overengineering
* не добавляй сложные паттерны без причины

---

### 2. Работай по шагам

* не генерируй весь проект сразу
* двигайся итерациями
* каждый шаг должен быть понятен

---

### 3. Разделяй ответственность

Используй структуру:

* routes → HTTP слой
* services → бизнес-логика
* schemas → валидация
* data → JSON
* utils → вспомогательное

---

### 4. Не ломай архитектуру

* не пихай всю логику в route.ts
* выноси в services
* избегай дублирования
* делай небольшие функции

---

### 5. Используй AI только где нужно

Используй LLM для:

* рекомендаций
* объяснений
* генерации текста

НЕ используй для:

* фильтрации
* простых условий
* вычислений

---

### 6. Работа с API

* выноси API в отдельные сервисы
* используй env для ключей
* обрабатывай ошибки
* делай fallback

---

### 7. Формат ответа с кодом

Когда генерируешь код:

1. кратко объясни
2. покажи полный файл
3. что изменилось
4. как проверить

---

### 8. Дебаг

Если есть ошибка:

* объясни причину
* покажи где проблема
* исправь код
* объясни как проверить

---

### 9. Ограничения

НЕ добавляй:

* auth
* базу данных
* оплату
* сложную архитектуру
* лишние зависимости

---

## Главная цель

Помочь сделать:

 простой
 понятный
 рабочий backend
 с хорошей архитектурой

А не перегруженный проект.
